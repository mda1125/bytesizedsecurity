# Contributing to ByteSizedSecurity

Thanks for your interest in contributing to ByteSizedSecurity! We welcome well-written articles on cybersecurity topics from the community.

## What We're Looking For

- **Security insights** ‚Äî Threat analysis, vulnerability breakdowns, defense strategies
- **Tutorials** ‚Äî Hands-on guides for tools, techniques, or concepts
- **Industry commentary** ‚Äî Perspectives on security news, trends, or best practices
- **DevSecOps** ‚Äî CI/CD security, automation, secure development practices

## How to Contribute

### Option 1: Submit via GitHub Issue

Open a new issue with the title: `[Blog Post] Your Article Title`

Format your issue body like this:

```
**Category:** (e.g., Pentesting, DevSecOps, Privacy, Threat Intel)

**Tags:** tag1, tag2, tag3

**Author:** Your Name

**Author Link (optional):** https://your-website-or-social

---

Your full article content in Markdown goes here.

Use proper headings (## for H2, ### for H3), code blocks, and lists.
```

We'll review your submission and, if accepted, convert it into a post with full attribution.

---

### Option 2: Submit via Pull Request

1. Fork this repository
2. Create a new file in `_posts/` with the naming format:
   ```
   YYYY-MM-DD-your-article-title.md
   ```
3. Use this frontmatter template:

   ```yaml
   ---
   layout: post
   title: "Your Article Title"
   category: Security
   tags: [Tag1, Tag2, Tag3]
   author: Your Name
   excerpt: "A brief 1-2 sentence summary of your article."
   ---
   ```

4. Write your content in Markdown below the frontmatter
5. Submit a pull request with a brief description

---

## Content Guidelines

- **Original content only** ‚Äî No plagiarism or AI-generated fluff
- **Actionable and educational** ‚Äî Readers should learn something useful
- **Well-structured** ‚Äî Use headings, lists, and code blocks appropriately
- **Respectful** ‚Äî No illegal activity promotion; ethical hacking only
- **Length** ‚Äî Aim for 300‚Äì1500 words (quality over quantity)

## Attribution

All accepted contributions receive **full author credit** displayed on the post. If you provide a link (website, Twitter/X, LinkedIn, GitHub), we'll include it with your byline.

Your work will be published under our [CC BY 4.0 license](LICENSE.md), meaning others can share it with attribution to you and ByteSizedSecurity.
